# ObsidianHub 产品需求文档（PRD）

## 1. 产品概述

ObsidianHub 是一个面向个人用户的自托管 Obsidian Server，核心目标是为个人 vault 提供：

- 远程访问
- 同步增强
- 多 agent 受控编辑
- 可审计、可回滚、可恢复的服务层

它不是 Obsidian 替代品，也不是团队知识库平台。

## 2. 目标用户

### 核心用户
- 使用 Obsidian 作为个人知识库的用户
- 有 VPS / NAS / 家庭服务器的自托管用户
- 希望让 AI / automation 参与笔记整理、索引、发布的高级用户

### 非目标用户
- 需要企业协作权限体系的团队
- 需要实时多人协作编辑的组织
- 想用它替代完整在线文档平台的用户

## 3. 用户问题

1. 不在本机时，访问 vault 不方便
2. 多个客户端 / 进程 / agent 同时操作 vault 时，容易产生冲突或覆盖
3. 现有同步方案通常不提供面向 agent 的受控 API
4. 让 agent 直接碰文件系统风险太高
5. 自动化改坏笔记后，缺少回滚与审计能力

## 4. 产品目标

### 必须达成
- 用户可以远程访问并查看自己的 vault
- agent 可以通过受控 API 操作笔记
- 关键写入具备冲突保护与恢复能力
- 部署与初始化足够简单，适合 Docker 场景

### 本阶段不追求
- 团队协作
- 复杂权限模型
- 实时协同编辑
- 大而全平台化

## 5. 核心场景

### 场景 A：远程查看与搜索
用户出门在外，通过浏览器查看笔记、搜索内容、下载附件。

### 场景 B：远程轻量编辑
用户在临时设备上对 note 做轻量修改，而不必完整同步整个 vault。

### 场景 C：agent 整理知识库
整理 agent 在受控目录中创建、重命名、补 frontmatter、归档笔记。

### 场景 D：agent 建立索引
索引 agent 读取全库内容，生成 tags、links、索引文件，但不能随意改正式笔记。

### 场景 E：错误恢复
某个 agent 或用户误改了大量内容，系统可基于快照或冲突副本恢复。
